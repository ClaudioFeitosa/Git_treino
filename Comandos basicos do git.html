comandos basicos do git

Git init

Este comando inicialisa seu reporsitorio, ele cria uma pasta git dentro da pasta do seu projeto Este comando você so vai utilizar uma vez por projeto não ha necessidade de fazer o git init mais de uma vez para o mesmo projeto


git add.

Este comando serve para adicionar os arquivos/modificações ele vai enviar os arquivos para Stage

git commit -m "mensagem explicativa"
git status

Este comando mostra o status do projeto, você pode usar ele a qq momento se usar logo apos o seu git add você vai ver a lista de arquivos que estão prontos para subir

git commit -m "mensagem explicativa"

Este comando serve para você colocar uma mensagem no pacote por exemplo indicar o que foi modificado neste pacote

git branch -M main

O comando git branch -M main é usado para renomear a branch padrão do repositório Git de "master" para "main". Este comando é útil quando você deseja adotar um vocabulário mais inclusivo e evitar termos que possam ter conotações negativas. não é necessario usar este comando a cada push que você fizer

git remote add origen git@github.com:claudiofeitosa/Git_treino.git

Este comando é usando na primeira vez que você iniciaçliza o reporsitorio ele serve para fazer o link entre seu repositorio local com o repositorio no git hub

git push -u origin main

git push -u origin main: A opção -u ou --set-upstream é usada para configurar a branch local para rastrear a branch remota. Isso significa que, após configurar o upstream (ligação entre a branch local e remota), você pode usar simplesmente git push nas próximas vezes sem especificar a branch e o repositório remoto. A sintaxe completa seria git push -u origin main para a configuração inicial.
